# AIO Framework — Ten Categories + Four Extensions + Industry Variants

Source: canonical AIO audit metadata at `ucp-analysis/apps/audit/src/data/audit-metadata.json` — **189 audits** across 10 categories. Moesica run surfaced 156; remaining 33 activate on certain page types (commerce, HowTo, local, authored content, pricing). Framework covers all 189.

Cross-referenced against the `agenticstorefront` AI Commerce Readiness audit engine (`AIAgentOptimizationService.ts`, 1901 LOC). Four extension categories (§11–14) and Industry Variants section added to cover signals that engine audits beyond the 189 canonical audits.

Weights reflect audit scoring model. Full coverage of weighted-high categories reliably scores above 80%.

## Category weights

| # | Category | Weight |
|---|---|---|
| 1 | Content Discoverability | 15% |
| 2 | AI Crawler Permissions | 8% |
| 3 | Structured Data & Schema | 12% |
| 4 | Meta Tags & AI Head Elements | 8% |
| 5 | AI Agent Tools & Action Surfaces | 18% |
| 6 | Semantic HTML & Content Structure | 8% |
| 7 | Accessibility & Agent Interaction | 7% |
| 8 | Technical Readiness & Security | 9% |
| 9 | Answer Engine Optimization | 7% |
| 10 | Generative Engine Optimization | 8% |

Readiness vitals the audit reports separately:
- **Commerce** — only relevant for stores.
- **Content** — fed by categories 1, 3, 9, 10.
- **AI Bot Accessibility** — fed by categories 2, 4, 8.
- **Technical Readiness** — fed by categories 5, 6, 8.

## 1. Content Discoverability (15%)

What an AI agent needs to find, index, and navigate your content without a full crawl.

### Required artifacts
- `/llms.txt` — H1 site name, blockquote summary, H2 sections with described links, including a link to `/llms-full.txt`.
- `/llms-full.txt` — full company context: positioning, every service, every page, FAQ in full, contact info, machine-readable resource index.
- `/sitemap.xml` — all non-excluded pages, absolute URLs, `<lastmod>`, `<changefreq>`, `<priority>`.
- `/sitemap-index.xml` — generated by `@astrojs/sitemap` integration.
- `/rss.xml` — full content (`<content:encoded>`), not just summaries, for every article-type page.
- `/robots.txt` — references both sitemap URLs and an `AI-Content-Map:` line pointing at `/llms.txt`.
- `/navigation.json` — machine-readable primary nav + machine-readable resource list.
- Per-page Markdown alternates (`/<path>/index.md`) via `@puralex/astro-markdown-for-agents`.

### Page-level requirements
- Single `<h1>` per page.
- No `noindex` on homepage or primary content.
- No `nofollow` on key internal links.
- `<link rel="canonical">` on every page.
- Mobile viewport meta (`width=device-width`).
- Internal linking: every page links to at least two others within the site; no orphans.
- No redirect chains (enforce single canonical trailing-slash via `.htaccess`).

### Astro wiring
- `astro.config.mjs` with `trailingSlash: "always"`, `build.format: "directory"`, `sitemap()` integration, `markdownForAgents()` integration.
- `src/pages/sitemap.xml.ts` as a secondary explicit sitemap for hard-coded routes (dynamic `[slug]` routes only appear in the generated sitemap index by default).
- `src/pages/rss.xml.ts` with `@astrojs/rss`, including `content:encoded` via `customData`/xmlns for full-article delivery.

## 2. AI Crawler Permissions (8%)

Explicit `User-agent: <bot>\nAllow: /` records tell AI crawlers they are welcome, even when the default policy is permissive. The audit warns when specific bots are not named.

### Required robots.txt directives

At minimum name each of the following with an explicit `Allow: /` block:

```
GPTBot, ChatGPT-User, OAI-SearchBot,
ClaudeBot, Claude-User, Claude-SearchBot, anthropic-ai,
Google-Extended,
PerplexityBot,
Applebot-Extended,
CCBot,
Meta-ExternalAgent, Meta-ExternalFetcher,
Amazonbot,
Bytespider,
cohere-ai,
YouBot,
Diffbot,
AI2Bot,
Bravebot,
DuckAssistBot,
MistralAI-User
```

### Additional requirements
- No blanket `User-agent: *\nDisallow: /`.
- No blocking of `/llms.txt`, `/sitemap.xml`, `/openapi.json`, `/mcp.json`, or any other AI-facing file.
- Protect sensitive paths explicitly (`Disallow: /admin/`, `/api/internal/`, etc.) rather than with a blanket.
- `Crawl-delay` either absent or ≤ 2.
- `<meta name="robots">` must not include `noindex` or `nofollow` on content pages.
- No aggressive bot-detection (Cloudflare "I'm under attack", etc.) that returns JavaScript challenges to AI bots.

See `templates/public/robots.txt` for the working version.

## 3. Structured Data & Schema Markup (12%)

JSON-LD on every page, validated, with cross-referenced `@id`s.

### Per-archetype requirements

| Archetype | Required types | Audits |
|---|---|---|
| Homepage | `Organization` (+ `ProfessionalService` or `LocalBusiness`), `WebSite` with `potentialAction: SearchAction`, `WebPage`, `OfferCatalog` (if services), `FAQPage` (if FAQ section), `SpeakableSpecification` | 3.1–3.4, 3.12, 3.7, 3.9 |
| Service page | `Organization` (ref), `Service`, `WebPage`, `BreadcrumbList`, `SpeakableSpecification`, `potentialAction` (ContactAction/OrderAction) | 3.3, 3.5, 3.8, 3.9, 3.10 |
| Service index | `CollectionPage`, `ItemList`, `BreadcrumbList` | 3.5 |
| Article / insight | `Organization` (ref), `Article` with `datePublished`, `dateModified`, `Person` author with `sameAs`, `WebPage`, `BreadcrumbList`, `SpeakableSpecification` | 3.5, 3.6, 3.15, 10.1, 10.2, 10.10 |
| Article index | `CollectionPage` with `mainEntity: ItemList`, `BreadcrumbList` | 3.5 |
| Author / bio page | `Person` with `jobTitle`, `sameAs` (LinkedIn/Twitter/GitHub), `affiliation` | 3.15, 10.2, 10.3 |
| How-to / tutorial | `HowTo` with ordered `HowToStep[]` | 3.11 |
| Confirmation / thank-you | `WebPage` with `potentialAction: ConfirmAction` | 3.16 |
| Local / physical business | `LocalBusiness` or `ProfessionalService` with `PostalAddress`, `telephone`, `openingHours` | 3.12 |
| Pricing page | `Offer` with `price`, `priceCurrency`, `availability`, `priceValidUntil` | 3.14 |
| Testimonial / review | `Review` with `Person` author and `Rating`, or `AggregateRating` with `ratingValue`, `reviewCount` | 3.13, 10.8 |
| Commerce product | `Product` with `name`, `description`, `sku`/`gtin13`/`mpn`, `brand`, `category`, `Offer`, `AggregateRating`, `Review` | 3.8, 3.13, 3.14, 3.21–3.23 |

### Cross-linking convention
All graphs use `@graph` with stable `@id`s:
- `${siteUrl}/#organization`
- `${siteUrl}/#website`
- `${canonicalHref}#webpage`
- `${siteUrl}/#services-catalog`
- `${canonicalHref}#service` or `#article`

Sub-entities reference by `@id` rather than duplicating. See `structured-data-patterns.md` for full graphs.

### Validation
- Must parse as JSON (validator fails on unescaped characters — use `JSON.stringify()` inside `<script type="application/ld+json" set:html={...}>`).
- Must pass Google Rich Results or Schema.org validator.
- `speakable.cssSelector` must resolve to elements that actually exist on the page.

## 4. Meta Tags & AI Head Elements (8%)

A `SeoHead.astro` component is the canonical place for this. One component, one prop surface.

### Required meta
- `<title>` — unique per page.
- `<meta name="description">` — unique per page, AEO formula (see category 9).
- `<meta name="author">` — named person or organization on every content page (audit 4.2 / GEO 10.1 E-E-A-T signal).
- `<meta name="viewport" content="width=device-width">`.
- `<html lang="en">` (or real content language).
- `<link rel="canonical">`.
- `<link rel="prev">` / `<link rel="next">` on paginated pages (audit 10.12).
- `<meta name="ai-content-declaration" content="<absolute llms.txt URL>">`.
- `<meta name="ai-instructions" content="<one-sentence agent hint for this page>">`.
- `<meta name="theme-color">`.
- `<meta name="generator">` (Astro emits automatically).
- `<meta name="robots">` not blocking except on utility pages (`thank-you`, `search`, etc.).

### Open Graph
- `og:title`, `og:type` (`website` or `article`), `og:description`, `og:url`, `og:site_name`.
- `og:image` on every shareable page, with `og:image:alt`.

### Twitter
- `twitter:card` = `summary_large_image` when image exists, otherwise `summary`.

### Head links (all absolute URLs)
- `<link rel="alternate" type="text/plain" href="/llms.txt">`
- `<link rel="alternate" type="text/plain" href="/llms-full.txt">`
- `<link rel="alternate" type="text/markdown" href="/path/index.md">` — per page markdown alternate.
- `<link rel="alternate" type="application/rss+xml" href="/rss.xml">`
- `<link rel="alternate" type="application/json" href="/ai-catalog.json">`
- `<link rel="alternate" type="application/json" href="/brand.json">`
- `<link rel="alternate" type="application/json" href="/openapi.json">`
- `<link rel="alternate" type="application/json" href="/mcp.json">`
- `<link rel="service-desc" type="application/openapi+json" href="/openapi.json">`
- `<link rel="author" type="text/plain" href="/humans.txt">`
- `<link rel="sitemap" type="application/xml" href="/sitemap.xml">` on at least the homepage.

See `templates/src/components/SeoHead.astro`.

## 5. AI Agent Tools & Action Surfaces (18% — highest weight)

This is the single biggest scoring lever. An AI agent should be able to discover every action and resource machine-readably.

### Required static files
- `/openapi.json` — OpenAPI 3.1 spec of every public action (contact form, search, etc.). Must have `servers`, `paths`, `operationId` per operation, request/response schemas. Add `x-ai-instructions` with a human-language summary.
- `/mcp.json` — MCP discovery manifest. Lists resources (URIs + MIME types) and tools (name, description, inputSchema, endpoint, method).
- `/ai-catalog.json` — flat index of `resources` and `actions` with type, title, url, description.
- `/.well-known/ai-plugin.json` — optional but recommended for ChatGPT plugin compatibility.
- `/agents.json` — optional agents manifest.
- WebMCP manifest — lists declarative tools for forms and actions on the site.

### On-page requirements
- Every CTA and form has `data-action`, `data-action-type` (`lead|navigation|content|commerce`), `data-action-label` attributes.
- Forms work without JavaScript (no JS-only validation blocking submit).
- Forms don't use CAPTCHA that blocks scripted clients (or provide a non-CAPTCHA alternative).
- At least one contact/lead endpoint exposed in OpenAPI.

### Astro wiring
- `public/openapi.json`, `public/mcp.json`, `public/ai-catalog.json` — static files.
- `SeoHead.astro` emits the head links to all four.
- Every `<a>` and `<button>` that fires a primary action gets the `data-action-*` triplet. See the homepage CTAs, header CTA, and form submit in the Moesica site.

## 6. Semantic HTML & Content Structure (8%)

### Required
- One `<h1>` per page (audit 6.1).
- Headings in sequence, no h2→h4 skips (audit 6.2). RAG systems use heading order for chunk boundaries.
- Landmarks: `<header>`, `<main id="main">`, `<footer>`, `<nav aria-label="...">` (audit 6.3, 6.5).
- Every self-contained content block wrapped in `<article>` (audit 6.4). RAG systems chunk on `<article>` boundaries for vector embedding. Applies to blog posts, news stories, product cards, forum posts.
- `<aside>` for sidebars, callouts, related-links, pull quotes (audit 6.6). Distinguishes supplementary content so it is not mixed into the main summary.
- Every `<section>` has a heading or `aria-label` / `aria-labelledby` (audit 6.7).
- Lists use `<ul>` / `<ol>` (audit 6.8). Use `<ol>` for any step-by-step content (pairs with AEO 9.6).
- Data tables use `<table>` with `<thead>`, `<tbody>`, `<th scope>` (audit 6.9). Comparison content should be a table (pairs with AEO 9.5).
- Code blocks tagged with `<pre><code class="language-xxx">` (audit 6.10). Prism/Highlight.js auto-emit this.
- Dates wrapped in `<time datetime="YYYY-MM-DD">Human date</time>` (audit 6.11). Every publication, modified, and event date. Pair with AEO 9.8 and GEO 10.9/10.10.
- Contact info wrapped in `<address>` — email, phone, physical address (audit 6.12).
- Glossaries / definitions use `<dl><dt><dfn>Term</dfn></dt><dd>Definition</dd></dl>` (audit 6.13). Use `<dfn>` inline for first-use term definitions.
- Every `<img>` has an `alt` (empty string for decorative) (audit 6.15, 6.16).
- `<figure>` + `<figcaption>` for meaningful image groups (audit 6.17).
- Page has substantial textual content (>300 words on key pages) (audit 6.14).

### Astro wiring
- Build semantic structure directly in `.astro` pages. Astro emits nothing extra.

## 7. Accessibility & Agent Interaction (7%)

### Required
- Skip-link at top of every page (`.skip-link` → `#main`).
- All form inputs have an associated `<label>`.
- All buttons and links have accessible text (no icon-only without `aria-label` or `.sr-only` text).
- Focus styles declared in CSS (visible `:focus-visible` outline).
- `prefers-reduced-motion` respected for any animations.
- Color contrast WCAG AA on body text.
- Modals use `role="dialog"` + focus trap (only if modals exist).
- Multiple navs distinguishable (`<nav aria-label="Primary">`, `<nav aria-label="Footer">`).

### Astro wiring
- `SiteHeader.astro` emits the `.skip-link` once.
- Use `variant` props to distinguish fixed vs. solid header rendering.

## 8. Technical Readiness & Security (9%)

### HTTPS + headers (set at server or CDN)
- HTTPS with HSTS: `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload`.
- `Content-Security-Policy` — tight default-src `'self'`, allow-listed external origins (fonts, form endpoint, image CDN).
- `X-Content-Type-Options: nosniff`.
- `X-Frame-Options: SAMEORIGIN`.
- `Referrer-Policy: strict-origin-when-cross-origin`.
- `Permissions-Policy: camera=(), microphone=(), geolocation=()` (or per-site).

### CORS
- `/llms.txt`, `/llms-full.txt`, `/robots.txt`, `/humans.txt`, `/sitemap*.xml`, `/rss.xml`, `/navigation.json`, `/ai-catalog.json`, `/brand.json`, `/openapi.json`, `/mcp.json`, `/.well-known/*` all served with `Access-Control-Allow-Origin: *`.
- `/.well-known/*` also serves `Access-Control-Allow-Methods: GET, OPTIONS`.

### Content-Types
- `.md` → `text/markdown`.
- `.json` → `application/json`.
- `.xml` → `application/xml`.
- `.txt` → `text/plain`.

### Cache
- AI-facing files (`llms*.txt`, `robots.txt`, `*.json`, `*.md`) → `Cache-Control: public, max-age=86400`.
- Hashed build output (`/_astro/*`) → `Cache-Control: public, max-age=31536000, immutable`.

### Server-rendered content
- The audit flags sites where the primary content only appears after JS hydration. Astro's static output satisfies this by default.
- Ensure no `<noscript>`-gated content on the critical path.

### Performance
- LCP element (hero image, hero text) is not lazy-loaded. Use `fetchpriority="high"` + explicit `width`/`height`.
- Explicit dimensions on every image.
- `preconnect` to image/font origins.
- No render-blocking resources beyond the stylesheet bundle.
- Compress text assets (`mod_deflate` or CDN).

### Privacy / compliance
- `/privacy/` and `/terms/` pages (link from footer). Add these even as stubs if the business has no formal policy yet.
- `/.well-known/security.txt` with `Contact:`, `Expires:`, `Preferred-Languages:`, `Canonical:`.

### Astro wiring
- Deploy with `public/.htaccess` (Apache) or translate to Nginx / `_headers` file (Netlify/Cloudflare Pages).
- Use `astro:assets` or explicit sized `<img>` tags. Keep hero `fetchpriority="high"`.

## 9. Answer Engine Optimization (7%)

Content is written to answer the user's question inside the page, in a way an AI can quote.

### Required patterns
- First paragraph of each page directly answers "what is this page" (audit 9.3).
- Meta description follows the formula: `<entity> is a <thing> that does <value>. It offers <specifics>.` Around 140–160 chars (audit 9.11).
- FAQ section with question-formatted headings on the home and on any page where users commonly ask questions (audit 9.1).
- Question-formatted H2/H3 headings ("What is X?", "How does X work?") (audit 9.2).
- Direct definitions for key terms (audit 9.4). Pair with `<dfn>`/`<dl>` markup (audit 6.13).
- Comparison tables as real `<table>` markup (audit 9.5). Convert feature/plan/tier comparison content from paragraphs to tables.
- Numbered steps as `<ol>` lists (audit 9.6). Procedural content in paragraph form loses step order in extraction.
- Specific numbers or timeframes wherever possible (audit 9.7). If no data exists yet, record a realistic placeholder the business owns.
- Visible publication date on every content page (audit 9.8, GEO 10.9). Use `<time datetime>` near the title/byline.
- Visible "Last updated" indicator (audit 9.10). Update date whenever the page content actually changes. Pair with `dateModified` in JSON-LD (audit 10.10).
- Every page answers its question without requiring a click-through to a subpage (audit 9.9).

### Astro wiring
- FAQs as a plain array in the page frontmatter, rendered into both the DOM and the `FAQPage` JSON-LD.
- Meta descriptions centralized in `src/data/*.ts` per page so they can be reviewed for AEO formula in one place.

## 10. Generative Engine Optimization (8%)

Signals that push LLMs to cite the site by name.

### Required
- **Named author attribution** on every content page (audit 10.1). Replace "Staff", "Admin", "Editor" with a real person name. Include `jobTitle` in JSON-LD. Generic authors collapse E-E-A-T trust scoring.
- **Author JSON-LD with `sameAs`** array linking to LinkedIn, Twitter/X, GitHub, personal site (audit 10.2). RAG systems use `sameAs` to cross-reference identity across platforms and build an expertise graph.
- **Author page per content creator** at `/authors/<slug>/` with bio, credentials, expertise areas, published articles (audit 10.3). Link from `Person.url` in every Article's `author` property.
- About page/section with credentials and experience (audit 10.4). Pair with `Organization` schema `founder`, `foundingDate`, `numberOfEmployees` where applicable.
- **External citations** — at least 2 outbound links per content page to authoritative sources (research, official docs, industry authorities) (audit 10.5). Pages without citations read as unsourced opinion to RAG trust scoring.
- Brand name appears in body text (not only in the logo) at least 2–3 times on the homepage (audit 10.6).
- Trust signals on homepage: client logos, testimonials, case metrics, certifications, press mentions (audit 10.7). Add what exists; do not fabricate.
- **Review / testimonial signals** — `Review` or `AggregateRating` JSON-LD, or attributed `<blockquote cite>` with `<cite>` and `<footer>` (audit 10.8).
- **Visible publication date** (`<time datetime>` near the title) (audit 10.9). Pair with `datePublished` in schema.
- **`dateModified` in Article / WebPage JSON-LD** distinct from `datePublished`, updated on revision (audit 10.10).
- Internal cross-linking (audit 10.11): every service page links to related services; every article links to at least two relevant services or articles.
- **Pagination links** `<link rel="prev">` + `<link rel="next">` in `<head>` of paginated listings (audit 10.12). Covered by `SeoHead.astro` `prevHref`/`nextHref` props.
- Unique data or statistics — the kind only this brand has (audit 10.13). Internal metrics, proprietary research, customer outcomes.
- Blockquotes / callouts for memorable lines (audit 10.14). LLMs preferentially cite highlighted text.
- Descriptive URL slugs (audit 10.15) — `/services/ai-transformation/`, not `/s/5/`.

### Astro wiring
- Data pattern: `src/data/services.ts`, `src/data/insights.ts` export typed arrays. Every slug is human-readable.
- Cross-links built from the data arrays so adding a new service automatically appears on related service pages.

## 11. Multi-Modal Capabilities (extension)

Beyond text, AI agents increasingly consume images, video, and audio. Extension category — not in canonical 189 but flagged by production audit engines.

### Required
- Every `<img>` has descriptive `alt` (>5 chars non-trivial) — decorative images use `alt=""`.
- Responsive images via `srcset` + `<picture><source>` for variant resolutions.
- Video elements include `<track kind="captions">` or `<track kind="subtitles">` with WebVTT.
- Audio elements include transcript (linked or inline).
- `ImageObject` JSON-LD on hero/product images with `contentUrl`, `width`, `height`, `caption`, `license`.
- `VideoObject` JSON-LD on any embedded video with `name`, `description`, `thumbnailUrl`, `uploadDate`, `duration` (ISO 8601).
- Downloadable PDFs / docs have machine-readable metadata (title, author in file properties).

### Astro wiring
- Use `astro:assets` `<Image />` for automatic `srcset` + `width`/`height`.
- Add `VideoObject` / `ImageObject` to the page's `structuredData` graph when media is present.

## 12. Voice / Conversational Optimization (extension)

Voice assistants (Alexa, Siri, Google Assistant) and conversational AI query sites the same way LLMs do but with stronger preferences for featured-snippet-shaped answers.

### Required
- Featured-snippet structures — `<ol>` for steps, `<ul>` for lists, `<table>` for comparisons. At least one per page where applicable.
- Conversational phrasing in body text where natural: "how to", "what is", "best way to", "near me", "open now", "reviews". Not keyword stuffing — natural language coverage.
- Question-formatted H2/H3 ("What is X?", "How does X work?") — pairs with audit 9.2.
- Brief direct answer within ~50 words after each question heading (voice assistants read out the first ~40 words).
- For local-business variant: visible `<address>`, `<time>` for hours, `tel:` links.

### Astro wiring
- Content discipline. Add a review step in content workflow — scan each page for at least one `<ol>` or `<table>` plus direct-answer opening.

## 13. Personalization Readiness (extension)

AI agents increasingly act on behalf of individual users. Signals that the site can accept and surface personalized state raise the likelihood of being chosen for agent-driven tasks.

### Required (when applicable — most relevant for SaaS / ecommerce)
- Cookie consent mechanism compliant with GDPR/CCPA — banner + granular preferences.
- Recommendation / related-content sections (`<aside>` with `aria-label`, cross-linked by content tags).
- User accounts + login system (`/login/`, `/account/`, `/signup/`).
- Behavioral tracking infrastructure (analytics, `data-track` attributes) — transparent and declared in the privacy policy.
- Content categorization — tags, categories exposed as URL paths (`/tags/<slug>/`) and in JSON-LD `keywords` / `about`.

### Out of scope
- Pure static marketing sites with no user accounts. Skip this category.

## 14. Real-Time Data & Freshness Signals (extension)

AI agents penalize stale data. Signals that content is live-updated raise trust for time-sensitive queries (pricing, availability, hours).

### Required
- `ETag` + `Last-Modified` response headers on HTML pages (Apache / CDN emits automatically when configured).
- `dateModified` in JSON-LD on every content page — must differ from `datePublished` when revised (audit 10.10).
- Visible "Last updated" `<time datetime>` near the top of content pages (audit 9.10).
- For live data (inventory, pricing, hours): expose via OpenAPI endpoint so agents can refresh without re-scraping HTML.
- WebSocket / SSE endpoint for real-time inventory / chat — document in OpenAPI if present.
- Cache-Control on AI files short enough to reflect updates (template uses `max-age=86400`, which is the right order of magnitude for daily-updated sites; use `max-age=3600` or lower for pricing pages).

### Astro wiring
- Enable Apache `FileETag` + `ExpiresByType` in `.htaccess`. Framework template sets long cache for hashed `_astro/` only — HTML and AI files are served with `ETag` by default.

## Industry Variants

The 10 base categories apply universally. Industry variant layer adds required artifacts, schemas, and pages per site type. Pick the one that fits and apply its extension on top of the base.

## Scoring leverage — order of implementation

Priority order for a fresh Astro site aiming to reach 80%+:

1. **Agent surface (cat 5)** — 18% weight, highest single return. OpenAPI + MCP + ai-catalog + navigation + `data-action` attributes.
2. **Content discoverability (cat 1)** — 15% weight. llms.txt, llms-full.txt, sitemap, RSS, per-page markdown alternates.
3. **Structured data (cat 3)** — 12% weight. JSON-LD graph on every archetype.
