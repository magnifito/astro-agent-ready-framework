# Templates

Copy-paste starters. Replace placeholders, then build.

## Placeholder reference

| Token | Replace with |
|---|---|
| `$SITE_NAME` | Human brand name, e.g. `Acme` |
| `$SITE_PASCAL_NAME` | PascalCase name used in operationIds, e.g. `Acme` |
| `$SITE_PACKAGE_NAME` | npm package name, e.g. `acme.com` |
| `$SITE_URL` | Absolute origin, no trailing slash, e.g. `https://acme.com` |
| `$SITE_DOMAIN` | Bare domain, e.g. `acme.com` |
| `$SITE_EMAIL` | Public business email, e.g. `hello@acme.com` |
| `$SITE_TAGLINE` | Short tagline |
| `$SITE_SUMMARY` | 1–2 sentence summary (used as meta description baseline) |
| `$SITE_LONG_DESCRIPTION` | Paragraph-length positioning |
| `$SITE_SHORT_PURPOSE` | Filler for `ai-instructions` default |
| `$SITE_POSITIONING` | 1-sentence positioning |
| `$PAGE_HOME_DESC` | llms.txt homepage description |
| `$RSS_DESCRIPTION` | Feed description |
| `$FORM_ENDPOINT` | Absolute URL the contact form POSTs to (e.g. Web3Forms `https://api.web3forms.com/submit`) |
| `$FORM_SERVER_URL` | Form processor origin without path (e.g. `https://api.web3forms.com`) |
| `$FORM_ACCESS_KEY` | Form processor public access key — the hidden `access_key` input on the homepage contact form (e.g. your Web3Forms key). Prompted by `init` |
| `$SITE_SNAKE_NAME` | snake_case name (derived by `init`), e.g. `acme` |
| `$SITE_LINKEDIN` / `$SITE_TWITTER` / `$SITE_GITHUB` | Optional social handles for `socialProfiles` (`sameAs`). Left in commented lines in `site.ts` — uncomment + fill only if used |
| `$UPDATED_ISO_DATE` | Today's date `YYYY-MM-DD` (set by `init`) |
| `$EXPIRES_ISO` | security.txt Expires datetime, e.g. `2027-04-15T00:00:00Z` (set by `init`) |
| `$INDEXNOW_KEY` | 32-hex IndexNow key (generated by `init`; also written to `public/<key>.txt`) |

### Content tokens — left as TODOs by `init`, fill with real copy before build

| Token | Replace with |
|---|---|
| `$PAGE_HOME_DESC` | llms.txt homepage description |
| `$RSS_DESCRIPTION` | Feed description |
| `$AUDIENCE_1/2/3` | Target audience bullets |
| `$FAQ_N_QUESTION/ANSWER` | FAQ entries |
| `$SERVICE_A/B_TITLE`, `$SERVICE_A/B_LONG_DESCRIPTION` | Service copy in llms-full.txt |
| `$SERVICE_ONE/TWO_{SLUG,TITLE,SUMMARY,BODY}` | Example service pages (`src/data/services.ts`) |
| `$INSIGHT_ONE/TWO_{SLUG,TITLE,DESCRIPTION,PUBLISHED,MODIFIED,TAG,BODY,PULLQUOTE}` | Example article pages (`src/data/insights.ts`) |
| `$AUTHOR_ONE_{SLUG,NAME,JOBTITLE,BIO,IMAGE,LINKEDIN,TOPIC}` | Author record (`src/data/authors.ts`) |
| `$CITATION_ONE..FOUR_{LABEL,URL}` | Insight citations / external references |
| `$PRIVACY_{INFORMATION_COLLECTED,INFORMATION_USE,THIRD_PARTIES,YOUR_RIGHTS}` | Privacy policy body (`src/pages/privacy.astro`) |
| `$TERMS_{USE_OF_SITE,INTELLECTUAL_PROPERTY,LIABILITY,CHANGES}` | Terms body (`src/pages/terms.astro`) |
| `$LEGAL_LAST_UPDATED` | ISO date on privacy/terms, e.g. `2026-01-01` |
| `$SITE_LONG_DESCRIPTION`, `$SITE_SHORT_PURPOSE`, `$SITE_POSITIONING` | Positioning prose |

> The build validator (`validate-built-site.js`) scans `dist/` for any leftover
> `$TOKEN` and **fails the build** — so every content token above must be
> resolved before shipping. `init` never fails on them; the build does.

## Suggested replace workflow

Copy the templates into your new site root, then run the bootstrap script — it
replaces the error-prone `sed` loops:

```bash
cd new-site-root
rsync -a /path/to/aio-framework/templates/ ./
npm install
npm run init
```

`npm run init` (`scripts/init.mjs`, zero-dependency Node) will:

1. **Collect your core values** — either interactively (it prompts for
   `SITE_NAME`, `SITE_URL`, `SITE_EMAIL`, tagline, summary, form endpoints, …
   and validates that `SITE_URL` is an absolute `https` origin with no trailing
   slash), or non-interactively from a `site.config.json` in the project root:

   ```json
   {
     "SITE_NAME": "Acme",
     "SITE_URL": "https://acme.com",
     "SITE_EMAIL": "hello@acme.com",
     "SITE_TAGLINE": "…",
     "SITE_SUMMARY": "…",
     "FORM_ENDPOINT": "https://api.web3forms.com/submit",
     "FORM_SERVER_URL": "https://api.web3forms.com"
   }
   ```

   Anything omitted is prompted for or derived. `SITE_DOMAIN`,
   `SITE_PASCAL_NAME`, `SITE_SNAKE_NAME` and `SITE_PACKAGE_NAME` are derived
   from the name/URL (each overridable); `UPDATED_ISO_DATE` / `EXPIRES_ISO` are
   set to today / today + 1 year; and an `INDEXNOW_KEY` is generated.

2. **Rewrite every `$TOKEN`** it has a value for across all text files (skipping
   `node_modules`, `dist`, `.git`, `.astro`). Re-running on an already-initialised
   tree is a safe no-op.

3. **Write the IndexNow key file** to `public/<INDEXNOW_KEY>.txt` (needed for
   `npm run indexnow`).

4. **Report remaining content tokens as a TODO list** — placeholders like
   `$FAQ_*`, `$SERVICE_*`, `$INSIGHT_*`, `$AUTHOR_*`, `$CITATION_*`,
   `$PRIVACY_*`, `$TERMS_*`, `$AUDIENCE_*`, `$PAGE_HOME_DESC` and
   `$RSS_DESCRIPTION` are left in place for you to fill in with real copy. Init
   prints each remaining token and the files it appears in, and never fails on
   them — edit those by hand, then `npm run verify`.
